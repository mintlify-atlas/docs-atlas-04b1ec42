adf-kit is a TypeScript library for working with **Auto-lead Data Format (ADF) 1.0** — the XML standard used by automotive CRMs, dealer management systems, and lead aggregators. It gives you a fluent builder API for generating leads, a lenient parser for inbound XML, a spec-compliant validator, and a serializer for round-tripping documents.

## What you can do with adf-kit

<CardGroup cols={2}>
  <Card title="Fluent builder API" icon="wand-magic-sparkles">
    `ADFProspectBuilder` and `ADFBuilder` let you construct valid ADF leads with TypeScript autocompletion — no need to read the spec.
  </Card>
  <Card title="Lenient XML parser" icon="diagram-project">
    `ADFparse` converts any ADF 1.0 XML string into a strongly-typed `ADFDocument` object, coercing values and filling defaults.
  </Card>
  <Card title="Spec-compliant validation" icon="list-check">
    `ADFvalidate` checks required fields, VIN format, ISO date strings, currency codes, country codes, and more — without throwing.
  </Card>
  <Card title="Zero runtime deps" icon="feather">
    The distributed bundle ships with no production dependencies. Fast installs, no supply-chain surprises.
  </Card>
</CardGroup>

## Quick example

```typescript
import { ADFProspectBuilder } from '@uhteddy/adf-kit';

const xml = new ADFProspectBuilder()
  .vehicle({
    year: 2025,
    make: 'Toyota',
    model: 'Camry',
    trim: 'XSE',
    interest: 'buy',
    status: 'new',
    price: 32_500,
  })
  .customer({
    firstName: 'Jane',
    lastName: 'Smith',
    email: 'jane.smith@example.com',
    phone: '555-867-5309',
  })
  .vendor({ name: 'Sunrise Toyota', email: 'leads@sunrisetoyota.com' })
  .string();
```

<Steps>
  <Step title="Install the package">
    ```bash
    npm install @uhteddy/adf-kit
    ```
  </Step>
  <Step title="Build a lead with the fluent API">
    Use `ADFProspectBuilder` to set the vehicle, customer, and vendor, then call `.string()` to get the XML output.
  </Step>
  <Step title="Parse inbound XML">
    Use `ADFparse` to convert incoming ADF XML strings into typed `ADFDocument` objects your CRM code can work with directly.
  </Step>
  <Step title="Validate before processing">
    Run `ADFvalidate` on any parsed document to get structured errors and warnings without throwing exceptions.
  </Step>
</Steps>
