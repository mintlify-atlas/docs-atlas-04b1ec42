---
title: "ADFProspectBuilder — fluent lead builder"
description: "Complete reference for ADFProspectBuilder: build a single ADF lead with a chainable TypeScript API, then export to XML string, Blob, or typed ADFProspect."
---

`ADFProspectBuilder` is the primary way to generate a single ADF 1.0 lead in adf-kit. It provides a fluent, chainable interface — you call `.vehicle()`, `.customer()`, and `.vendor()` in any order, then call a terminal method (`.string()`, `.xml()`, or `.build()`) to produce output. The TypeScript compiler enforces that all three required fields are set before the terminal methods become available, so missing data is caught at compile time rather than at runtime.

## Constructor

```typescript
import { ADFProspectBuilder } from '@uhteddy/adf-kit';

const builder = new ADFProspectBuilder();
```

`ADFProspectBuilder` takes no arguments. All configuration is done through chained method calls.

---

### `.customer(input)`

### `.requestDate(date)`

Overrides the lead timestamp. When not called, the timestamp defaults to the current time at the moment `.build()`, `.string()`, or `.xml()` is invoked.

```typescript
builder.requestDate(date: Date | string): ADFProspectBuilder
```

<ParamField path="date" type="Date | string" required>
  The lead request date and time. Accepts a JavaScript `Date` object or an ISO 8601 date/datetime string.
</ParamField>

### `.status(s)`

Marks the lead as a resend of a previously submitted lead. Defaults to `"new"` when not called.

```typescript
builder.status(s: 'new' | 'resend'): ADFProspectBuilder
```

<ParamField path="s" type="'new' | 'resend'" required>
  Use `"resend"` when retransmitting a lead that was already sent. Use `"new"` (the default) for first-time submissions.
</ParamField>

## Terminal methods

<Note>
  The terminal methods `.build()`, `.string()`, and `.xml()` are only available after `.vehicle()`, `.customer()`, and `.vendor()` have all been called. The TypeScript compiler enforces this — attempting to call a terminal method before setting all three required fields produces a compile-time type error.
</Note>

### `.build()`

Returns the fully-typed `ADFProspect` object without serializing. Use this when you want to inspect or transform the data before converting it to XML, or when you are passing the object to another adf-kit function such as `ADFvalidate`.

```typescript
builder.build(): ADFProspect
```

### `.string()`

Serializes the lead to an ADF 1.0 XML string. Use this when your HTTP client, email library, or file writer accepts a raw string body.

```typescript
builder.string(): string
```

### `.xml()`

Serializes the lead to a `Blob` with `Content-Type: application/xml`. The Blob carries the MIME type automatically, so you can pass it directly to `fetch` or any library that accepts `BodyInit`, without setting headers manually.

```typescript
builder.xml(): Blob
```

## Examples

<Tip>
  You can call `.vehicle()`, `.customer()`, and `.vendor()` in any order — all three return the same builder instance. Only the terminal methods require all three to be set.
</Tip>

### New car purchase lead

```typescript new car purchase lead
import { ADFProspectBuilder } from '@uhteddy/adf-kit';

const xml = new ADFProspectBuilder()
  .vehicle({
    year: 2025,
    make: 'Toyota',
    model: 'Camry',
    trim: 'XSE',
    interest: 'buy',
    status: 'new',
    price: 32_500,
    colors: [{ exterior: 'Midnight Black', interior: 'Black' }],
  })
  .customer({
    firstName: 'Jane',
    lastName: 'Smith',
    email: 'jane.smith@example.com',
    phone: '555-867-5309',
    address: { city: 'Austin', regionCode: 'TX', postalCode: '78701' },
    timeFrame: { description: 'Within 30 days', earliest: '2025-05-01', latest: '2025-05-31' },
  })
  .vendor({ name: 'Sunrise Toyota', email: 'leads@sunrisetoyota.com', phone: '512-555-0100' })
  .provider({ name: 'AutoTrader', service: 'New Car Search', url: 'https://autotrader.com' })
  .string();
```

## Output method comparison

<Tabs>
  <Tab title=".string()">
    Returns the ADF XML as a plain string. Use with HTTP clients or libraries that accept a string body, or when you want to write the XML to a file.

    ```typescript
    const xml: string = new ADFProspectBuilder()
      .vehicle({ year: 2025, make: 'Toyota', model: 'Camry' })
      .customer({ firstName: 'Jane', email: 'jane@example.com' })
      .vendor({ name: 'Sunrise Toyota', email: 'leads@sunrisetoyota.com' })
      .string();

    // Use as a fetch body with a manual Content-Type header
    await fetch(endpoint, {
      method: 'POST',
      headers: { 'Content-Type': 'application/xml' },
      body: xml,
    });
    ```
  </Tab>
  <Tab title=".xml()">
    Returns the ADF XML as a `Blob` with `Content-Type: application/xml` already set. Pass it directly to `fetch` or any `BodyInit`-compatible library — no need to set the MIME header yourself.

    ```typescript
    const blob: Blob = new ADFProspectBuilder()
      .vehicle({ year: 2025, make: 'Toyota', model: 'Camry' })
      .customer({ firstName: 'Jane', email: 'jane@example.com' })
      .vendor({ name: 'Sunrise Toyota', email: 'leads@sunrisetoyota.com' })
      .xml();

    // Content-Type is carried by the Blob automatically
    await fetch(endpoint, { method: 'POST', body: blob });
    ```
  </Tab>
  <Tab title=".build()">
    Returns the fully-typed `ADFProspect` object without serializing to XML. Useful when you need to inspect values, pass the object to `ADFvalidate`, or feed it into `ADFBuilder` alongside other prospects.

    ```typescript
    import { ADFProspectBuilder, ADFvalidate } from '@uhteddy/adf-kit';
    import type { ADFProspect } from '@uhteddy/adf-kit';

    const prospect: ADFProspect = new ADFProspectBuilder()
      .vehicle({ year: 2025, make: 'Toyota', model: 'Camry' })
      .customer({ firstName: 'Jane', email: 'jane@example.com' })
      .vendor({ name: 'Sunrise Toyota', email: 'leads@sunrisetoyota.com' })
      .build();

    const result = ADFvalidate({ version: '1.0', prospects: [prospect] });
    if (!result.valid) {
      console.error(result.errors);
    }
    ```
  </Tab>
</Tabs>
