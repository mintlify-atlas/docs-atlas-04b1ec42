---
title: "Get started with adf-kit in 4 steps"
description: "Install adf-kit, generate your first ADF lead with ADFProspectBuilder, parse inbound ADF XML with ADFparse, and validate a document with ADFvalidate."
---

This page covers the four essential operations in adf-kit: installing the package, building a spec-compliant ADF lead from scratch, parsing inbound ADF XML into typed TypeScript objects, and validating a parsed document against the ADF 1.0 specification. Each step is self-contained — skip ahead to whichever operation is most relevant to your use case.

<Steps>
  <Step title="Install adf-kit">
    Add `@uhteddy/adf-kit` to your project using your preferred package manager. TypeScript 5+ and an ESM module configuration are required.

    ```bash
    npm install @uhteddy/adf-kit
    ```

    Verify the install by importing the main builder:

    ```typescript
    import { ADFProspectBuilder } from '@uhteddy/adf-kit';
    console.log(typeof ADFProspectBuilder); // "function"
    ```

    For full setup instructions including `tsconfig.json` requirements, see [Installing adf-kit](/installation).
  </Step>

  <Step title="Generate your first lead">
    Use `ADFProspectBuilder` to construct a new car purchase lead. Chain `.vehicle()`, `.customer()`, `.vendor()`, and optionally `.provider()`, then call `.string()` to get the ADF XML.

    ```typescript
    import { ADFProspectBuilder } from '@uhteddy/adf-kit';

    const xml = new ADFProspectBuilder()
      .vehicle({
        year: 2025,
        make: 'Toyota',
        model: 'Camry',
        trim: 'XSE',
        interest: 'buy',
        status: 'new',
        price: 32_500,
        colors: [{ exterior: 'Midnight Black', interior: 'Black' }],
      })
      .customer({
        firstName: 'Jane',
        lastName: 'Smith',
        email: 'jane.smith@example.com',
        phone: '555-867-5309',
        address: { city: 'Austin', regionCode: 'TX', postalCode: '78701' },
        timeFrame: { description: 'Within 30 days', earliest: '2025-05-01', latest: '2025-05-31' },
      })
      .vendor({ name: 'Sunrise Toyota', email: 'leads@sunrisetoyota.com', phone: '512-555-0100' })
      .provider({ name: 'AutoTrader', service: 'New Car Search', url: 'https://autotrader.com' })
      .string();
    ```

    `xml` is now a complete ADF 1.0 XML string ready to POST to a dealer CRM endpoint or save to a file. To get a `Blob` with `Content-Type: application/xml` instead — useful as a `fetch` body or email attachment — call `.xml()` in place of `.string()`.
  </Step>

  <Step title="Parse inbound ADF XML">
    Use `ADFparse` to convert an ADF XML string received from a lead provider into a typed `ADFDocument`. Access vehicle, customer, and vendor data as structured TypeScript properties.

    ```typescript
    import { ADFparse } from '@uhteddy/adf-kit';

    const doc = ADFparse(xmlString);
    // doc.prospects[0].vehicles[0].make  → "Toyota"
    // doc.prospects[0].customer.contact.emails[0].value  → "jane@example.com"
    ```

    To iterate over all leads in a document:

    ```typescript
    import { ADFparse } from '@uhteddy/adf-kit';

    const doc = ADFparse(xmlString);

    for (const prospect of doc.prospects) {
      const { customer, vendor, vehicles, requestDate, status } = prospect;

      const name = customer.contact.names.find(n => n.part === 'first')?.value;
      const email = customer.contact.emails[0]?.value;
      const phone = customer.contact.phones[0]?.value;

      for (const vehicle of vehicles) {
        console.log(`${requestDate} | ${status} | ${vehicle.year} ${vehicle.make} ${vehicle.model}`);
        console.log(`  Customer: ${name} <${email}> ${phone}`);
        console.log(`  Dealer:   ${vendor.vendorName}`);
      }
    }
    ```

    `ADFparse` throws an `ADFParseError` if the XML is malformed or missing required elements. Wrap the call in a `try/catch` in production code.
  </Step>

  <Step title="Validate a document">
    Use `ADFvalidate` to check a parsed document against the ADF 1.0 specification. It never throws — it always returns a `ValidationResult` with `valid`, `errors`, and `warnings` fields.

    ```typescript
    import { ADFparse, ADFvalidate } from '@uhteddy/adf-kit';

    const doc = ADFparse(xmlString);
    const result = ADFvalidate(doc);

    if (!result.valid) {
      for (const issue of result.errors) {
        console.error(`[${issue.path}] ${issue.message}`);
      }
    }

    for (const warn of result.warnings) {
      console.warn(`[${warn.path}] ${warn.message}`);
    }
    ```

    `errors` are spec violations — missing required fields, invalid contact information, or out-of-range values — that indicate the document does not conform to ADF 1.0. `warnings` flag non-fatal issues such as malformed VINs, suspect vehicle years, or invalid currency codes that may cause downstream problems without strictly violating the spec.
  </Step>
</Steps>
